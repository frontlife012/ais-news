# Ai's News サイト公開手順(GitHub Pages)

所要時間の目安: 15〜20分。すべて無料です。
手順1〜3でニュースサイトが公開され、手順4で掲示板が使えるようになります。

---

## 手順1: GitHubにリポジトリを作る

1. https://github.com/new を開く
2. **Repository name** に `ais-news` と入力
3. **Public** を選択(Pagesの無料利用にはPublicが必要)
4. ほかは何も変えずに **Create repository** をクリック

## 手順2: コードをアップロード(プッシュ)する

このフォルダでターミナル(PowerShell)を開き、以下を1行ずつ実行します。
`あなたのユーザー名` は自分のGitHubユーザー名に置き換えてください。

```
git remote add origin https://github.com/あなたのユーザー名/ais-news.git
git push -u origin main
```

※初回はGitHubへのログイン画面が出るので、ブラウザで認証してください。
※コミットは作成済みなので、上の2行だけでアップロードできます。

## 手順3: GitHub Pagesを有効にする

1. GitHubのリポジトリページ → **Settings** → 左メニューの **Pages**
2. **Source**: 「Deploy from a branch」を選択
3. **Branch**: 「main」、フォルダは「**/docs**」を選択して **Save**
4. 1〜2分待つと、ページ上部に公開URLが表示されます:
   **`https://あなたのユーザー名.github.io/ais-news/`**

### 自動更新について

- リポジトリの **Actions** タブを開き、有効化を求められたら有効にしてください
- 以後、**6時間ごとに自動でニュースが収集・翻訳されてサイトに反映**されます
- すぐ更新したいときは Actions → **Update news** → **Run workflow** で手動実行できます

## 手順4: 掲示板を有効にする(Supabase・無料)

掲示板を使わない場合はこの手順は不要です(サイト上では「準備中」と表示されます)。

1. https://supabase.com で無料アカウントを作成し、**New project** を作成
   (Database Passwordは適当な強いパスワードでOK。Regionは「Northeast Asia (Tokyo)」推奨)
2. 左メニューの **SQL Editor** を開き、下のSQLを貼り付けて **Run**:

```sql
create table posts (
  id uuid primary key default gen_random_uuid(),
  name text,
  body text not null check (char_length(body) <= 1000),
  topic text not null default 'general',
  likes int not null default 0,
  created_at timestamptz not null default now()
);

alter table posts enable row level security;

create policy "誰でも閲覧可" on posts
  for select to anon using (true);

create policy "誰でも投稿可" on posts
  for insert to anon
  with check (char_length(body) <= 1000 and coalesce(char_length(name), 0) <= 24);

create or replace function like_post(post_id uuid)
returns int language sql security definer as $$
  update posts set likes = likes + 1 where id = post_id returning likes;
$$;

grant execute on function like_post(uuid) to anon;
```

3. 左メニューの **Project Settings** → **API** を開き、次の2つをコピー:
   - **Project URL**(例: `https://abcdefgh.supabase.co`)
   - **anon public** キー(長い文字列)
4. このフォルダの [docs/config.js](docs/config.js) を開き、2つの値を貼り付けて保存:

```js
window.AISNEWS_CONFIG = {
  SUPABASE_URL: 'https://abcdefgh.supabase.co',
  SUPABASE_ANON_KEY: 'eyJhbGciOiJIUzI1NiIs...'
};
```

5. 変更をアップロード:

```
git add docs/config.js
git commit -m "掲示板を有効化"
git push
```

1〜2分後にサイトの掲示板が使えるようになります。

※anon publicキーはサイト利用者に見える前提で設計された公開用キーなので、
config.jsに書いて公開しても問題ありません(書き込み権限はSQLのポリシーで制限済み)。

## よくある質問

**Q. 更新の頻度を変えたい**
[.github/workflows/update-news.yml](.github/workflows/update-news.yml) の `cron: '15 */6 * * *'` を変更
(例: `'0 */3 * * *'` で3時間ごと、`'0 21 * * *'` で毎朝6時(日本時間))

**Q. 手元で動作確認したい**
このフォルダで `npm start` → http://localhost:3456

**Q. タブ(ニュースソース)を増やしたい**
[sources.js](sources.js) に追記してプッシュすれば、次の自動更新から反映されます

**Q. 不適切な掲示板投稿を消したい**
SupabaseのダッシュボードのTable Editor → postsテーブルから該当行を削除
